## 一、系統邏輯大綱

### 1.1 系統定位

本系統為**多館別飯店暨不動產綜合管理 ERP**，整合以下業務線：

| 業務線 | 說明 |
|--------|------|
| 飯店營運 | PMS 日報收入、房間統計、住客人次 |
| 採購進貨 | 廠商、進貨單、庫存、付款 |
| 短租（民宿） | Airbnb / Booking 收入帳務 |
| 長租（商辦） | 租約、租金收款、押金、稅費 |
| 不動產資產 | 房屋、土地、折舊、租金損益 |
| 工程管理 | 專案合約、材料、廠商付款 |
| 帳務財務 | 現金流、支票、貸款、月結 |

---

### 1.2 核心概念

#### 館別（Warehouse）
系統以「館別」區分不同營運地點，所有收支、庫存、現金帳戶皆可分館管理。常見館別如：麗格、麗軒、民宿。

#### 作業流程（付款流程）
```
進貨單 / 費用單
  ↓ 建立
付款單（Finance）
  ↓ 審核送出
出納確認（Cashier）
  ↓ 執行
現金流交易紀錄（Cashflow）
  ↓ 月底
銀行對帳（Reconciliation）
  ↓ 確認無誤
月結關帳（Month-End）
```

#### 期間鎖定
月結關帳後，該月的所有交易記錄**鎖定不可修改**。如需解鎖必須由有權限的人員填寫原因，並留下稽核紀錄。

#### 稽核日誌
系統所有**新增、修改、刪除**操作皆自動記錄操作人、時間、異動前後資料，符合財務合規要求。

---

### 1.3 資料流程圖

```
【外部資料來源】
  PMS 系統 ──┐
  Airbnb    ──┤──→ 收入記錄（各模組）
  人工輸入   ──┘

【費用端】
  進貨單 ──→ 付款單 ──→ 出納執行 ──→ 現金帳扣款
  費用模板 ─┘
  工程付款 ─┘

【收入端】
  PMS 收入 ──→ 推送至現金流 ──→ 現金帳入帳
  租金收款 ──┘

【分析層】
  現金流 + 收入 + 支出 ──→ 月結報告 ──→ 分析儀表板
```

---

### 1.4 月結作業週期

每月底須依序完成：

1. **確認所有進貨單** 均已入庫或作廢
2. **確認發票** 已核銷或標記
3. **PMS 收入推送** 至現金流
4. **銀行對帳** 確認帳戶餘額一致
5. **現金盤點** 完成月底盤點
6. **月結關帳** → 系統自動產生月結報告

## 二、使用者角色與權限

### 2.1 預設角色說明

| 角色 | 主要職責 | 可存取模組 |
|------|---------|-----------|
| **管理員 (admin)** | 系統管理、使用者設定 | 全部 |
| **採購人員 (purchasing)** | 進貨、廠商、庫存 | 採購、庫存、廠商、產品 |
| **銷售人員 (sales)** | 發票、銷貨 | 銷售、發票 |
| **財務人員 (finance)** | 付款單、月結 | 財務、出納、現金流、月結 |

前往系統網址 → 輸入帳號（Email）與密碼 → 登入後依角色自動顯示可存取的功能。

## 三、儀錶板（首頁）

**路徑：`/`（首頁）**

儀錶板是系統進入點，提供整體營運狀況一覽。

### 主要區塊

| 通知代碼 | 意義 |

## 四、採購與庫存

### 4.1 採購管理 `/purchasing`

**用途：** 對廠商下進貨單、追蹤入庫狀態。

#### 操作流程

```
1. 點擊「新增進貨單」
2. 選擇廠商 → 選擇館別/部門
3. 加入品項（可搜尋產品主檔）
4. 儲存 → 狀態變為「待入庫」
5. 貨到後 → 點擊「確認入庫」→ 庫存自動增加
6. 建立對應付款單（可一鍵送至財務）
```

#### 狀態說明

| 狀態 | 說明 |
|------|------|
| 草稿 | 尚未送出 |
| 待入庫 | 已送出，等待驗收 |
| 已入庫 | 貨品已入庫，庫存已更新 |
| 已作廢 | 取消此進貨單 |

---

### 4.2 庫存管理 `/inventory`

**用途：** 查詢即時庫存、處理入庫/領料/調撥。

#### 五大功能頁籤

| 頁籤 | 功能 |
|------|------|
| **庫存查詢** | 依產品/館別查詢目前庫存量與金額 |
| **待入庫** | 進貨單確認入庫的作業入口 |
| **領料申請** | 部門申請出庫（減少庫存） |
| **調撥作業** | 館別間庫存轉移 |
| **盤點作業** | 實際點數與系統核對，記錄差異 |

---

### 4.3 產品主檔 `/products`

**用途：** 維護所有可採購產品的基本資料。

#### 欄位說明

| 欄位 | 說明 |
|------|------|
| 產品代碼 | 唯一識別碼 |
| 類別 | 例：食材、備品、工具 |
| 單位 | 例：箱、個、公斤 |
| 成本價 | 採購參考定價 |
| 售價 | 銷售參考定價 |
| 列入庫存 | 勾選後需指定倉庫位置 |
| 會計科目 | 連結至會計科目主檔 |

---

### 4.4 廠商主檔 `/suppliers`

**用途：** 維護合作廠商基本資料與付款條件。

#### 重要欄位

| 欄位 | 說明 |
|------|------|
| 統一編號 | 公司稅號（可用於開立發票） |
| 付款條件 | 月結 30/60/90 天、即付等 |

## 六、財務付款

### 6.1 付款管理 `/finance`

**用途：** 建立付款單、審核後送出納執行。

#### 付款流程

```
進貨單 / 費用 → 建立付款單 → [財務審核] → 送出 → 出納執行
```

#### 付款單狀態

| 狀態 | 說明 |
|------|------|
| 草稿 | 尚未送出審核 |
| 待出納 | 已送出，等待出納執行 |
| 已執行 | 出納已完成付款 |
| 已作廢 | 取消此付款單 |

#### 注意事項
- 建立付款單時必須選擇付款方式（現金/支票/轉帳/貸款帳）。
- 支票付款需填寫支票資訊（票號、到期日、銀行）。

### 6.2 會計科目 `/accounting-subjects`

**用途：** 維護財務分類帳（會計科目主檔）。

科目分四層：**分類 → 類別 → 代碼 → 名稱**

例：`資產 → 流動資產 → 1101 → 現金`

> 一般作業人員不需修改此頁面，由財務主管或系統管理員維護。

## 七、出納作業

### 7.1 出納 `/cashier`

**用途：** 出納人員執行財務已核准的付款單。

#### 三個工作區

| 頁籤 | 說明 |
|------|------|
| **待執行** | 待出納確認的付款清單 |
| **已執行** | 歷史執行記錄 |
| **已退回** | 被出納退回的付款單 |

#### 執行操作

1. 在「待執行」清單找到要處理的付款單
2. 確認金額、付款方式無誤
3. 點擊「確認執行」
4. 系統自動：
   - 建立現金流支出記錄
   - 扣減對應現金帳戶餘額
   - 若為支票：建立支票記錄

#### 出納退回
若付款資訊有誤，可點擊「退回」並填寫原因，付款單退回財務重新修改。

## 九、銀行對帳

> **重要：本章有兩套對帳模組，用途不同，請勿搞混。**

| 模組 | 路徑 | 用途 | 月結關聯 |
|------|------|------|---------|
| **存簿對帳** | `/reconciliation` | 信用卡帳單、OTA 佣金等科目對帳 | 月結 checklist 項目 |
| **存簿核對** | `/bank-reconciliation` | 銀行存款帳戶月結調節表（銀行月結對帳） | 月結 checklist 項目；**年結前 12 月必須完成** |

> 現金流餘額對得上，**不代表**銀行月結調節表已完成。年結前應確認 12 月存簿核對狀態。

---

### 9.1 存簿對帳 `/reconciliation`

**用途：** 將信用卡帳單、OTA 傭金等外部對帳單與系統記錄逐筆配對，確認科目一致。適用於非銀行存款帳戶的對帳。

#### 操作流程

```
1. 從信用卡公司或 OTA 平台下載對帳單（CSV 格式）
2. 匯入系統
3. 系統自動配對相符交易（金額+日期）
4. 手動確認無法自動配對的項目
5. 完成後確認餘額一致
```

#### 配對狀態

| 狀態 | 說明 |
|------|------|
| 已配對 | 銀行記錄與系統記錄相符 |
| 待確認 | 需人工確認 |
| 差異 | 金額或日期不一致，需調查 |

---

### 9.2 存簿核對（銀行調節表） `/bank-reconciliation`

**用途：** 銀行存款帳戶月結對帳——將銀行寄送的月結單與系統現金流餘額逐項勾稽，建立調節表。此為年結的前置條件。

#### 與存簿對帳的差異

- 存簿對帳（`/reconciliation`）處理信用卡／OTA 等科目配對，不直接影響年結
- 存簿核對（`/bank-reconciliation`）處理銀行存款帳戶調節表，**12 月份未完成將擋住年結**

#### 操作流程

```
1. 從銀行網站下載月結單（PDF 或 XLS）
2. 至「存簿核對」選擇帳戶與月份
3. 逐筆核對系統交易與銀行月結單
4. 加入調節項目（在途款、未兌票等）
5. 確認調節後餘額與銀行月結餘額一致
6. 標記為「已確認」
```

#### 年結前注意

- 月結 checklist 會每月提醒當月存簿核對狀態
- 年結前必須完成所有銀行帳戶的 **12 月份**存簿核對，否則年結將被擋住
- 完成後期初餘額才能準確結轉至下一年度

## 十、支票管理

**用途：** 管理所有應付/應收支票，追蹤到期與兌現狀態。

#### 五大頁籤

| 頁籤 | 說明 |
|------|------|

#### 支票狀態

| 狀態 | 說明 |
|------|------|
| 待兌現 | 尚未到期 |

#### 重要提醒
儀錶板通知會在支票到期前 **7 天**自動提醒。退票發生時會觸發**緊急通知**。

#### 五大頁籤

#### 貸款類型

| 類型 | 說明 |
|------|------|

## 十二、費用管理

### 12.1 費用 `/expenses`

**用途：** 管理定期重複性費用（如租金、管理費、清潔費），支援範本快速執行。

#### 三大功能

| 頁籤 | 功能 |
|------|------|
| **費用範本** | 建立可重複使用的費用範本（含分錄設定） |
| **快速執行** | 選擇月份，一鍵套用範本建立當月費用 |
| **執行記錄** | 查閱歷史費用執行與支付狀態 |

#### 費用範本設定
範本可設定：
- 費用名稱、分類、館別
- 借貸分錄（複式記帳）
- 預計採購品項

```

5. 旅宿網申報：填寫並列印申報資料
6. 鎖帳此月：月鎖＝整月不可改；筆鎖（全部鎖帳）＝單筆付款鎖定
```

> 月結 checklist 會自動檢查：各館是否已鎖帳、出納同步是否失敗。

#### 鎖帳說明

- **月鎖**（鎖帳此月）：整月所有訂房記錄、付款明細、匯入、申報均不可修改。
- **筆鎖**（全部鎖帳 🔒）：僅鎖定個別已填付款記錄，不影響月鎖狀態，兩者獨立。

#### 注意事項

- 訂金匯款核對後系統自動寫入訂房後五碼備查。
- 刷卡入帳日未填時，訂房明細頂部會出現紫色警告橫幅。
- 已退房但未填款的訂單，頂部會出現橘色警告橫幅並連結至付款稽核。

## 二十一、月結與年結

### 21.1 月結 `/month-end`

**用途：** 執行每月財務關帳，鎖定該月所有交易紀錄。

#### 月結前置確認清單（系統自動檢查）

| 檢查項目 | 說明 |
|---------|------|
| 逾期待入庫進貨單 | 超過 30 天未入庫的進貨單 |
| 逾期待核銷發票 | 超過 60 天未核銷的發票 |
| 未完成付款 | 仍在「待出納」狀態的付款單 |
| 現金帳戶餘額不一致 | 系統計算餘額與帳戶餘額不符 |
| 現金盤點未完成 | 月底盤點尚未確認 |
| 館別未完成個別月結 | 若有分館管理，各館應先各自月結 |

#### 月結狀態

| 狀態 | 說明 |
|------|------|
| 未結帳 | 預設狀態，可任意修改 |
| 已結帳 | 月結完成，產生月結報告 |
| 已鎖定 | 完全鎖定，需解鎖才能修改 |

#### 月結解鎖
如需修正已結帳月份資料，需填寫**解鎖原因**，解鎖操作會記錄在稽核日誌，並通知管理員。

---

### 21.2 年結 `/year-end`

**用途：** 完成整年度帳務收尾，將各帳戶目前餘額結轉為下一年度的期初餘額，並產生年度財務摘要報告。

#### 年結前置條件（blockers）

年結執行前系統自動驗證以下條件，任一未完成將擋住年結：

| 條件 | 說明 |
|------|------|
| 全館 12 個月月結完成 | 每個館別每個月份都必須是「已結帳」或「已鎖定」 |
| 全館月結（無館別）完成 | 全館層級的 12 個月也需完成 |
| VAT 第 6 期（11–12 月）已計算 | 留抵稅額需確認後方可結轉 |
| **12 月銀行帳戶存簿核對完成** | 所有銀行存款帳戶 12 月份調節表需已確認，否則期初餘額不準確 |

> 詳細的 12 月存簿核對流程請見第 9 章（[存簿核對](#九銀行對帳)）。

#### 年結執行步驟

```
1. 確認所有月份均已月結（月結管理查看）
2. 完成 12 月銀行存簿核對
3. 至「年結」頁面選擇年份
4. 點「驗證」— 確認所有 blocker 均已通過
5. 點「執行年結」— 系統將各帳戶 currentBalance 寫入下年度 openingBalance
6. 下載並保存年度財務摘要報告
```

#### 年結後注意事項

年結完成後，去年所有月份仍保持「已結帳/已鎖定」狀態，系統會擋住任何去年交易的修改。

**如需修改去年帳務：**

1. **月結管理解鎖** — 至 `/month-end` 解鎖對應月份（需填寫解鎖原因，留稽核記錄），修改後重新月結並鎖定
2. **沖銷流程** — 金額調整可在現金流模組對原交易執行「沖銷」，產生對衝交易（不需解鎖月份）
3. **勿直接改期初餘額** — 年結後勿手動修改帳戶期初餘額，應透過正式沖銷或年結重跑

> 沖銷不支援移轉類型交易；移轉交易若需調整，請解鎖月份後刪除重建。

## 二十三、常見問題 FAQ

**Q1：為什麼我看不到某些功能？**
A：每個功能依角色權限控管。請聯絡管理員確認您的角色設定。

**Q2：現金帳戶餘額和實際不符，怎麼辦？**
A：到 `/cashflow` → 帳戶總覽，系統可自動重算餘額（根據所有已確認交易）。若仍有差異，請到「對帳作業」逐筆核對。

**Q3：月結後發現資料有誤，可以修改嗎？**
A：需先由有權限人員執行「解鎖」操作（需填理由），修改後重新月結。操作全程記錄稽核日誌。

**Q4：如何設定 LINE 通知？**
A：到 `/settings` → 通知設定 → 掃描 QR Code 與 LINE 帳號綁定。通知內容可依個人偏好設定。

**Q5：PMS 資料要怎麼上傳？**
A：到 `/pms-income` → 收入記錄 → 點擊「匯入」→ 選擇 PMS 匯出的 CSV 檔案，確認欄位對應後上傳。每日最少上傳一次。

**Q6：進貨單被退回怎麼辦？**
A：出納退回後，付款單會回到「草稿」狀態，確認退回原因後修改再重新送出。

**Q7：忘記密碼怎麼辦？**
A：聯絡系統管理員重設密碼。目前系統無自助重設密碼功能。

**Q8：資料可以匯出嗎？**
A：大多數列表頁面右上角有「匯出」按鈕，可匯出當前篩選條件的資料為 Excel/CSV 格式。

## 二十四、常見踩雷（實務）

以下為實務中最常見的操作失誤，每一條都曾造成帳務混亂或對帳困難，請特別留意。

| 踩雷 | 後果 | 建議 |
|------|------|------|
| 出納已執行又在現金流手動記一筆 | 帳上重複計費，影響損益報表 | 勿在現金流手動補登；應至出納紀錄確認付款狀態 |
| 支票付款又兌現時多記一筆 | 帳上重複計費，影響損益報表 | 兌現只需在對帳頁面確認，勿再手動新增交易 |
| 在現金流改自動交易金額 | 系統拒絕或帳務不一致 | 回來源單據（付款單／進貨單）修改後重新執行 |
| 月結後補登上月交易 | 系統擋單無法儲存 | 先由有權限人員解鎖該月並填寫原因，修改後重新月結 |
| 只月結全館、各館未結 | 月結 checklist 出現警告，無法完成全館月結 | 多館環境須先逐館月結，再執行全館月結 |
| 損益科目長期未分類 | 損益報表科目空白或歸錯類，數字失真 | 每月使用批次歸類功能補齊科目分類 |
| 代墊款／信用卡當一般銀行對帳 | 對帳流程走錯，餘額無法軋平 | 依帳戶類型選擇正確對帳方式（員工代墊／信用卡各有專屬頁籤） |
| 職責不分（同一人建單又出納） | 內控失效、舞弊風險高、稽核無據 | 財務建單與出納執行**必須由不同人擔任**；管理員設定角色時確認職責分離，參考[角色設定](#二使用者角色與權限) |
