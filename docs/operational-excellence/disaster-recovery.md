# 災難復原與營運目標（Disaster Recovery）

## 1. RPO / RTO（請依公司實際填寫）

| 項目 | 定義 | 目標值（範例） | 實際達成方式 |
|------|------|----------------|--------------|
| **RPO** | 最多可接受遺失的資料時間 | 例如 24 小時 | 每日全庫備份 + 必要時 WAL/雲端 PITR |
| **RTO** | 最多可接受服務中斷時間 | 例如 4 小時 | 還原演練熟練度 + Staging 驗證 |

> 老闆與 IT 需共同確認上表數字，並每年檢討一次。

## 3. 災難等級與動作

| 等級 | 情境 | 建議動作 |
|------|------|----------|
| L1 | 單一 API 錯誤、可快速回滾版本 | 回滾部署、檢查 log |
| L2 | DB 不可用但備份完好 | 切換唯讀或維護頁 → 新庫還原 → 切回 |
| L3 | 資料中心 / 帳號外洩 | 啟動完整 DR、旋轉所有秘密、通報 |

## 4. 溝通範本（簡訊 / LINE）

- 「系統維護中，預計 X 小時內恢復；緊急請洽 ___」  
- 「已確認資料還原至 YYYY-MM-DD HH:mm 備份點，請業務抽樣驗證」

## 5. 相依清單（檢查清單）

- [ ] 正式 `DATABASE_URL` 與備份來源一致且加密傳輸  
- [ ] 附件儲存（若用 S3/R2）有版本或週期快照  
- [ ] DNS / SSL 憑證到期日有監控  
- [ ] 第三方（LINE、郵件、銀行）失敗時有替代聯絡管道
