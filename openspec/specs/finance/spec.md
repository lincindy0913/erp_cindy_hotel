# 財務管理 (Finance)

## Purpose

管理付款紀錄、支出對帳與付款憑單（傳票）的完整財務流程，支援多張發票合併付款、支票資訊管理、會計折讓、自動建立支出記錄、付款狀態追蹤，以及付款憑單的歷史價格比較功能。

## Requirements

### Requirement: 付款紀錄管理
系統 SHALL 提供付款紀錄（Payment）完整管理功能，支援多張發票合併付款、支票資訊、會計折讓。

付款紀錄資料模型包含：付款編號（唯一自動產生）、關聯發票 ID 列表（JSON 陣列）、付款日期、付款方式、金額、折讓、狀態、支票開票日、支票日期、支票號碼、開票賬戶、備註。

#### Scenario: 建立付款紀錄
- **WHEN** 使用者勾選一或多張未付款發票並送出付款
- **THEN** 系統驗證每張發票存在且未被重複付款
- **AND** 自動產生付款編號，格式為 `YYYYMM + 3位序號`（如 `202602001`）
- **AND** 金額計算：若使用者未提供 amount，系統以 `發票總金額 - 折讓` 計算
- **AND** 在交易（$transaction）內：建立付款記錄 → 每張發票狀態更新為「已核銷」→ 為每張發票建立支出記錄

#### Scenario: 重複付款防止
- **WHEN** 使用者嘗試對已付款的發票再次付款
- **THEN** 系統掃描所有現有付款記錄的 invoiceIds
- **AND** 若發現重複，回傳 400 錯誤

#### Scenario: 付款後自動建立支出
- **WHEN** 付款建立成功
- **THEN** 系統為每張關聯發票自動建立 Expense 記錄
- **AND** 支出記錄包含發票號碼、金額、廠商資訊（從進貨主檔反查）、館別

#### Scenario: 查詢付款列表
- **WHEN** 使用者查詢付款列表
- **THEN** 系統回傳所有付款記錄，依 ID 升冪排序
- **AND** 額外計算 `salesId`（取 invoiceIds 第一筆）

#### Scenario: 付款狀態行內編輯
- **WHEN** 使用者在列表中點擊付款狀態 badge
- **THEN** 系統提供行內下拉切換「未完成」/「已完成」
- **AND** 即時更新狀態

#### Scenario: 展開付款詳情
- **WHEN** 使用者展開付款記錄
- **THEN** 系統顯示付款資訊（方式、支票資料、折讓等）
- **AND** 顯示所支付的發票詳情表格

#### Scenario: 付款金額自動計算
- **WHEN** 使用者勾選或取消勾選發票
- **THEN** 系統自動計算 `發票總金額 - 會計折讓 = 付款金額`

#### Scenario: 自動帶入上次付款資訊
- **WHEN** 使用者開啟付款表單
- **THEN** 系統自動帶入最近一筆付款的開票日期和支票日期

### Requirement: 付款方式管理
系統 SHALL 支援多種付款方式並提供選項管理功能。

#### Scenario: 付款方式選項
- **WHEN** 使用者選擇付款方式
- **THEN** 系統提供可管理的選項列表
- **AND** 支援新增/刪除自訂選項

#### Scenario: 支票付款資訊
- **WHEN** 付款方式涉及支票
- **THEN** 系統提供支票相關欄位：開票日期、支票日期、支票號碼、開票賬戶
- **AND** 開票賬戶提供可搜尋的下拉選單

### Requirement: 支出管理
系統 SHALL 提供支出（Expense）管理功能，追蹤發票對帳與實際付款情況。

支出資料模型包含：關聯發票 ID、發票號碼、發票日期、金額、實付日期、實付金額、狀態、廠商 ID/名稱、館別。

#### Scenario: 自動建立支出
- **WHEN** 付款流程完成
- **THEN** 系統自動為每張發票建立支出記錄
- **AND** 初始狀態為「未完成」，實付日期為空，實付金額為 0

#### Scenario: 手動建立支出
- **WHEN** 使用者手動建立支出記錄（必填：invoiceId、invoiceNo、amount）
- **THEN** 系統建立支出記錄

#### Scenario: 行內編輯支出
- **WHEN** 使用者在支出列表中點擊編輯
- **THEN** 實付日期和實付金額欄位變為可編輯的 input
- **AND** 支援儲存/取消

#### Scenario: 查詢支出列表
- **WHEN** 使用者查詢支出
- **THEN** 系統回傳所有支出記錄，依 ID 升冪排序

#### Scenario: 刪除支出
- **WHEN** 使用者刪除支出記錄
- **THEN** 系統移除該記錄

### Requirement: 付款憑單（傳票）
系統 SHALL 提供付款憑單查看功能，顯示發票的完整進貨溯源與歷史價格比較。

#### Scenario: 查看付款憑單
- **WHEN** 使用者指定發票 ID 查看付款憑單（`/api/payment-voucher/{invoiceId}`）
- **THEN** 系統回傳：廠商名稱、館別、發票摘要（號碼/日期/金額/稅/總金額/狀態）
- **AND** 每個品項包含：產品資訊、數量、單價、小計、進貨單溯源資訊

#### Scenario: 歷史價格比較
- **WHEN** 系統組裝付款憑單品項
- **THEN** 對每個品項查詢該產品的全部歷史價格
- **AND** 計算歷史最低價（排除與當前價格相同的記錄）
- **AND** 若當前價格高於歷史最低價，標記 `isPriceHigher = true` 並計算價差 `priceDifference`

#### Scenario: 付款憑單列表頁
- **WHEN** 使用者瀏覽付款憑單列表頁
- **THEN** 系統顯示所有發票列表
- **AND** 支援銷帳年月、廠商、館別篩選
- **AND** 每張發票提供「列印傳票」連結（新視窗開啟）

### Requirement: 未付款發票篩選
系統 SHALL 提供付款頁面的未付款發票篩選功能。

#### Scenario: 篩選未付款發票
- **WHEN** 使用者在付款頁面指定銷帳年月、廠商、館別、付款條件
- **THEN** 系統查詢並顯示符合條件的未付款發票
- **AND** 支援多張發票勾選合併付款
